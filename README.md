# Интернет-магазин Laplandia (Лапландия)

Этот проект представляет собой интернет-магазин одежды/товаров с полноценным разделением на Frontend (на React + Vite) и Backend (на чистом PHP). Также в проекте настроена автоматическая синхронизация категорий и товаров из внешней системы учета (поставщика) через YML/XML-выгрузку.

---

## 📂 Структура проекта

*   **`backend/`** — серверная часть проекта на PHP.
    *   `config.php` — конфигурация подключения к базе данных MySQL.
    *   `includes/` — вспомогательные скрипты (например, подключение к БД в `db.php`).
    *   `api/` — API-эндпоинты, возвращающие данные в формате JSON (авторизация, заказы, товары, категории).
    *   `api/.env` — переменные окружения для работы с JWT, SMTP-сервером почты (PHPMailer) и CORS.
    *   `scripts/` — служебные скрипты (скрипт импорта товаров `import_products.php`).
*   **`frontend/`** — клиентская часть на React + Vite.
    *   `src/` — исходный код SPA-приложения.
        *   `api/config.js` — конфигурация адреса API.
        *   `components/` — переиспользуемые UI-компоненты (шапка, подвал, галерея изображений, форма заказа).
        *   `context/` — контексты управления состоянием авторизации (`AuthContext.jsx`) и корзины (`CartContext.jsx`).
        *   `pages/` — основные страницы интернет-магазина (Главная, Товары, Категории, Детали товара, Корзина, Мои заказы, Авторизация).
        *   `styles/` — стили страниц и компонентов, написанные на чистом CSS (с использованием CSS Modules).
    *   `server.js` — простой Express-сервер для раздачи статической сборки (dist) в production-режиме.
*   **`table.md`** — файл со схемой базы данных MySQL.

---

## 🛠️ Технологический стек

### Frontend:
*   **Библиотека**: React 18
*   **Сборщик**: Vite 6
*   **Маршрутизация**: React Router DOM 7
*   **Запросы**: Axios
*   **Уведомления**: React Toastify
*   **Иконки**: React Icons

### Backend:
*   **Язык**: PHP (Vanilla)
*   **Авторизация**: JWT (JSON Web Tokens) с библиотекой `firebase/php-jwt`
*   **Отправка почты**: PHPMailer (для верификации email и сброса пароля)
*   **Управление конфигурацией**: `vlucas/phpdotenv`
*   **База данных**: MySQL

## 🚀 Инструкция по запуску

### 1. Подготовка Базы Данных (MySQL)
1.  Создайте новую базу данных в MySQL (например, с именем `shop_db`).
2.  Откройте файл [table.md](file:///c:/Program%20Files/Ampps/www/laplandia/table.md) и выполните содержащиеся там SQL-запросы в вашей СУБД для создания необходимых таблиц (`categories`, `products`, `product_images`, `users`, `orders`, `order_items`).

### 2. Настройка и запуск Backend
1.  **Настройка подключения к БД**:
    Откройте файл [backend/config.php](file:///c:/Program%20Files/Ampps/www/laplandia/backend/config.php) и укажите ваши параметры подключения:
    ```php
    define('DB_HOSTNAME', 'localhost');
    define('DB_USERNAME', 'root');
    define('DB_PASSWORD', 'ваш_пароль_mysql');
    define('DB_DATABASE', 'shop_db');
    ```
2.  **Настройка окружения (.env)**:
    Перейдите в папку `backend/api` и откройте файл [backend/api/.env](file:///c:/Program%20Files/Ampps/www/laplandia/backend/api/.env). Укажите в нем ваши настройки SMTP-сервера (для отправки писем подтверждения почты и сброса пароля), а также секретный ключ для шифрования JWT-токенов:
    ```env
    MAIL_USERNAME="ваша_почта@gmail.com"
    MAIL_PASSWORD="ваш_пароль_приложений"
    ADMIN_EMAIL="ваша_почта@gmail.com"
    MAIL_HOST="smtp.gmail.com"
    MAIL_PORT=587
    MAIL_ENCRYPTION="STARTTLS"
    JWT_SECRET_KEY="ваш_секретный_ключ"
    FRONTEND_URL="http://localhost:5173"
    BACKEND_URL="http://localhost:3000"
    ```
3.  **Установка зависимостей**:
    Откройте терминал в папке `backend/` и выполните установку библиотек через Composer:
    ```bash
    composer install
    ```
4.  **Развертывание**:
    Поскольку бэкенд написан на чистом PHP, его необходимо запускать на веб-сервере (например, Apache в AMPPS, XAMPP, OpenServer). Убедитесь, что папка проекта `laplandia` находится в корневой директории веб-сервера (например, `www/` или `htdocs/`).
    *   Адрес API будет доступен по пути: `http://localhost/laplandia/backend/api/` (или скорректируйте в зависимости от настроек виртуального хоста).

### 3. Настройка и запуск Frontend
1.  **Установка зависимостей**:
    Перейдите в папку `frontend/` в терминале и установите JS-зависимости:
    ```bash
    npm install
    ```
2.  **Настройка адреса API**:
    Откройте файл [frontend/src/api/config.js](file:///c:/Program%20Files/Ampps/www/laplandia/frontend/src/api/config.js) и настройте URL вашего PHP-бэкенда:
    ```javascript
    export const API_URL = 'http://localhost/laplandia/backend/api/';
    ```
3.  **Запуск в режиме разработки**:
    Выполните команду для запуска dev-сервера Vite:
    ```bash
    npm run dev
    ```
    Приложение запустится по адресу: `http://localhost:5173` (или указанному в терминале).
4.  **Сборка и запуск в Production-режиме**:
    Для сборки оптимизированного бандла и его запуска через Express:
    ```bash
    npm run prod
    ```
    Сборка будет доступна на порту, указанном в Express-сервере (по умолчанию `http://localhost:3000`).

## 🔄 Описание работы скрипта импорта (`import_products.php`)

Файл [import_products.php](file:///c:/Program%20Files/Ampps/www/laplandia/backend/scripts/import_products.php) — это автоматический скрипт синхронизации данных, предназначенный для импорта категорий, товаров и их изображений из внешнего источника (поставщика/платформы MyDrop) через API-выгрузку формата YML (Yandex Market Language / XML).

### Ключевые функции скрипта:

1.  **Загрузка данных**:
    *   Скрипт делает HTTP-запрос по указанному API URL для получения XML-файла с актуальными остатками и ценами товаров.
    *   Парсинг структуры XML выполняется с помощью встроенной библиотеки PHP `SimpleXML`.

2.  **Синхронизация категорий (Добавление и обновление)**:
    *   Проходит в цикле по списку категорий в YML-файле.
    *   Проверяет существование родительских категорий в таблице `categories`. Если родительская категория отсутствует в базе данных, скрипт автоматически создает её, чтобы не нарушать целостность связей (внешних ключей).
    *   Если категория еще не добавлена в локальную базу данных, скрипт выполняет ее вставку (`INSERT`) с сохранением связей `parent_id`.

3.  **Синхронизация товаров**:
    *   Скрипт проходит в цикле по списку товаров (`<offer>`) из XML-файла.
    *   Извлекает основные параметры товара: уникальный идентификатор `id`, `group_id`, ID категории `category_id`, название, подробное описание, цену, флаг доступности (`available`), остаток на складе (`quantity_in_stock`), вес, а также ищет пользовательский параметр с размером одежды (параметр с атрибутом `name="Размер"`).
    *   **Проверка и логика сохранения**:
        *   **Если товара нет в базе данных**: Выполняется добавление (`INSERT`) нового товара в таблицу `products`.
        *   **Если товар уже существует**: Скрипт сверяет текущую цену и количество товара на складе. Если цена или остаток изменились, скрипт обновляет (`UPDATE`) эти данные в БД, чтобы на сайте всегда отображалась актуальная информация.
    *   **Импорт медиафайлов**:
        *   Для каждого товара скрипт проходит по всем тегам картинок `<picture>` и сохраняет внешние URL-адреса изображений в связанную таблицу `product_images`.

4.  **Статистика работы**:
    *   По завершении выполнения скрипт выводит подробную статистику в браузер/консоль:
        *   Общее количество обработанных товаров в выгрузке.
        *   Количество добавленных товаров.
        *   Количество обновленных товаров (у которых изменилась цена или остаток).
        *   Уведомление об успешном завершении импорта.

### 🕒 Как запускать импорт:
Импорт можно запустить вручную, открыв файл в браузере по адресу:
`http://localhost/laplandia/backend/scripts/import_products.php`

Для автоматизации обновления цен и наличия товаров рекомендуется настроить **Планировщик задач (Cron)** на сервере для вызова этого PHP-скрипта с заданной периодичностью (например, раз в час или раз в сутки):
```bash
0 * * * * php /путь_к_проекту/backend/scripts/import_products.php > /dev/null 2>&1
```
