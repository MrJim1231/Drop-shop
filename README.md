# 🛒 DropShop — Повноцінний Інтернет-Магазин (MERN Stack + TS)

🌐 **Live Demo:** [https://drop-shop-front.onrender.com](https://drop-shop-front.onrender.com)

Сучасний і швидкий інтернет-магазин постільної білизни та товарів, переписаний на сучасний стек технологій: **Node.js / Express / MongoDB (Mongoose) backend** та **React + TypeScript + Tailwind CSS v4 SPA client**.

Проект підтримує повноцінну систему кошика (включаючи кастомні простирадла на гумці), авторизацію користувачів та кабінет замовлень (для зареєстрованих користувачів та гостей), адмін-панель із завантаженням Excel/XML каталогів товарів, встановленням знижок та управлінням даними.

## 🛠️ Технологічний стек

- **Frontend**: React 19, TypeScript, Tailwind CSS v4, Vite, Axios, React Router, Lucide React, React Toastify.
- **Backend**: Node.js, Express, MongoDB (Mongoose ODM), JWT (jsonwebtoken), bcryptjs, Multer (завантаження файлів), Nodemailer (відправка пошти через SMTP), XLSX (парсинг таблиць), fast-xml-parser.
- **База даних**: MongoDB (локальна або MongoDB Atlas).

## ✨ Ключові можливості та оптимізації

1. **Миттєве завантаження категорій (Кешування та Паралелізм)**:
   - Переписано дерево категорій на бэкенді з позбавленням від проблеми N+1 запитів. Всі підкатегорії вибираються одним запитом.
   - Реалізовано вбудований в оперативну пам'ять кеш зображень-заглушок (`fallbackImageCache`) та паралельний запуск промісів через `Promise.all`. Завантаження головної сторінки прискорилося в десятки разів (з 14 секунд до менш ніж 0.1 секунди).
2. **Преміальна темна тема (Dark Mode)**:
   - Перемикач теми інтегровано в шапку сайту. Стан теми синхронізується з `localStorage`.
   - Використовується селекторний режим Tailwind CSS v4, який повністю незалежний від системних налаштувань ОС користувача.
3. **Управління акціями та знижками**:
   - Адміністратор може призначати відсоток знижки (до 90%) на будь-який товар через адмін-панель.
   - Акційні товари виводяться на сторінці `/deals` із перекресленою старою ціною та відсотковим бейджем.
4. **Консоль логів імпорту в реальному часі (Streaming Logs)**:
   - При запуску імпорту через адмін-панель бэкенд стрімить HTML-текст логу виконання чанками через `Transfer-Encoding: chunked`.
   - Адміністратор бачить реальний прогрес імпорту у фреймі-терміналі прямо на сторінці адмінки.
5. **Гнучкий кошик**:
   - Можливість додати опцію «Простирадло на гумці» для будь-якого товару з автоматичним перерахунком вартості.

## 🚀 Запуск проекту

### 1. Налаштування змінних середовища (.env)

Створіть файл `.env` всередині папки **`backend/`** та заповніть наступні змінні:

```env
PORT=5000
MONGO_URI="mongodb+srv://<username>:<password>@cluster.mongodb.net/drop-shop?retryWrites=true&w=majority"
JWT_SECRET="ваш_секретний_ключ_для_токенів"

# Налаштування SMTP пошти (для верифікації та надсилання замовлень)
EMAIL_USER="ваша_пошта@gmail.com"
EMAIL_PASS="пароль_додатку_google"

CLIENT_URL="http://localhost:5173/"
BASE_URL="http://localhost:5000"
```

> **Підказка:** Для відправки пошти через Gmail увімкніть [двоетапну перевірку Google](https://myaccount.google.com/security) та згенеруйте **Пароль додатка** (App Password).

### 2. Встановлення залежностей

У кореневій директорії проекту (де лежить загальний `package.json`) виконайте:

```bash
npm install
```
Ця команда автоматично встановить залежності запуску в корінь. Для встановлення залежностей клієнта і сервера перейдіть у відповідні папки:
```bash
cd backend && npm install
cd ../frontend && npm install
```

### 3. Одночасний запуск клієнта та сервера

Для запуску фронтенду (порт `5173`) та бэкенду (порт `5000`) однією командою, виконайте в корені проекту:

```bash
npm start
```

## 📥 Імпорт товарів до бази даних

### Спосіб А: Через консоль (Node.js скрипти)

У проекті є готові консольні скрипти для швидкого наповнення бази даних MongoDB:

1. **Імпорт з Excel файлу (`catalog_dropt_2026-07-12.xlsx` в корені)**:
   ```bash
   # Повний імпорт з очищенням старих товарів і категорій:
   node backend/scripts/import_products.js --reset

   # Імпорт із додатковою націнкою 15%:
   node backend/scripts/import_products.js --markup=15
   ```
2. **Імпорт з XML/YML фід посилання**:
   ```bash
   node backend/scripts/import_xml.js --url="https://example.com/feed.xml" --reset
   ```

### Спосіб Б: Через панель адміністратора (UI)

1. Увійдіть під обліковим записом адміністратора (email: `berolegnik@gmail.com` або `test@example.com`).
2. Перейдіть до розділу **Адмінка** -> таб **Імпорт та статистика**.
3. Виберіть та завантажте Excel-файл каталогу.
4. Натисніть кнопку **Імпортувати** та спостерігайте за логом обробки у реальному часі.
